### Elevation

- **Level 0 (bg):** `#131313` — sidebar, app background
- **Level 1 (surface):** `#201f1f` — cards, inputs, panels
- **Level 2 (overlay):** `#2a2a2a` — toasts, tooltips
- **Borders:** `1px solid #554434`
- **Focus glow:** `0 0 0 4px rgba(255, 152, 0, 0.15)`
- **Hover glow:** primary elements get +10% brightness; secondary get 10% orange bg tint

## Component Architecture

All new files go under `src/components/`.

```
src/
├── app/
│   ├── globals.css          ← Replace tokens, add typography/elevation classes
│   ├── layout.tsx           ← No change needed
│   └── page.tsx             ← Refactored: import components, keep state + API logic
│
└── components/
    ├── ui/                   ← Reusable primitives
    │   ├── Button.tsx        ─ 3 variants: primary, secondary, danger
    │   ├── Card.tsx          ─ Wrapper with panel bg + border + padding
    │   ├── Tag.tsx           ─ Pill chip, label-sm style
    │   ├── Toast.tsx         ─ Auto-dismiss status bar
    │   └── MetricCard.tsx    ─ Stat display (number + label)
    │
    ├── Sidebar.tsx           ─ Logo, tagline, 5 nav buttons
    ├── TopBar.tsx            ─ 3 MetricCards + Refresh btn
    ├── PracticeView.tsx      ─ Question, textarea, answer key, critique, self-grade
    ├── DraftsView.tsx        ─ Draft card list + approve/reject
    ├── NotesView.tsx         ─ Note list + generate/sync buttons
    ├── HistoryView.tsx       ─ Review list with rating badges
    └── SettingsView.tsx      ─ Settings form
```

### State/API Logic Stays in `page.tsx`

`page.tsx` remains the root container with:
- `useState` for view routing, all data
- `api()` helper
- All `async handle*` functions (handleSaveSettings, handleSyncNotion, etc.)
- View-to-component mapping via `view` state
- Status toast state and lifecycle

Each view component receives props (data + handlers) — no internal state.

## Implementation Steps (Ordered)

### 1. Update globals.css

- Swap color variables to Orange/Geist values
- Add typography classes (`.display-lg`, `.headline-md`, `.body-lg`, `.label-md`, `.label-sm`, `.mono`)
- Import or `@font-face` Geist and JetBrains Mono fonts
- Change `body` font-family to `Geist`
- Add `.focus-ring` utility (orange outer glow)
- Add elevation classes (`.level-0`, `.level-1`, `.level-2`)
- Add button hover/active glow states
- Reduce border-radius from 8px to 4px everywhere
- Add `.empty-state` centered styling
- Add view transition animation
- Fix spacing to 8px grid units (`--space-*` variables)

### 2. Create UI Primitives (`src/components/ui/`)

Each is a small pure component:

- **Button.tsx** — `<Button variant="primary|secondary|danger" />` - handles hover glow, focus ring, disabled
- **Card.tsx** — `<Card>...</Card>` — panel bg + border + padding wrapper
- **Tag.tsx** — `<Tag label="..." />` — pill chip with label-sm style
- **Toast.tsx** — renders status bar with auto-dismiss, success/error variants
- **MetricCard.tsx** — `<MetricCard value={n} label="..." />` — top bar stat

### 3. Create View Components

Each component receives typed props:

| Component | Props |
|-----------|-------|
| `Sidebar` | `{ view: ViewType, onViewChange: (v: ViewType) => void }` |
| `TopBar` | `{ stats: { dueCount, draftCount, reviewCount }, onRefresh: () => void }` |
| `PracticeView` | `{ activeCard, userAnswer, setUserAnswer, showAnswerKey, setShowAnswerKey, aiCritique, onCritique, onReview }` |
| `DraftsView` | `{ drafts: Draft[], onApprove: (id) => void, onReject: (id) => void }` |
| `NotesView` | `{ notes: Note[], onGenerate: (id) => void, onSync: () => void }` |
| `HistoryView` | `{ reviews: Review[] }` |
| `SettingsView` | `{ settings: Settings, onSave: (e) => void }` |

### 4. Refine Screens via Stitch MCP

Use `stitch_edit_screens` to update the 5 main screens with the Orange/Geist

### 5. Refactor page.tsx

Replace all inline JSX sections with imported components:

```tsx

import TopBar from '@/components/TopBar';

import NotesView from '@/components/NotesView';
import HistoryView from '@/components/HistoryView';
import SettingsView from '@/components/SettingsView';

return (
  <main className="shell">
    <Sidebar view={view} onViewChange={setView} />
    <section className="content">
      <TopBar stats={stats} onRefresh={loadState} />
      {status && <Toast message={status.message} isError={status.isError} />}
      {view === 'practice' && <PracticeView | ... />}
      {view === 'drafts'   && <DraftsView   ... />}
      {view === 'notes'    && <NotesView    ... />}
      {view === 'history'  && <HistoryView  ... />}
      {view === 'settings' && <SettingsView ... />}
    </section>
  </main>
);
```
